# Next Permutation

## Problem Statement
Given a collection of distinct integers, return all possible next lexicographic permutations of the given integers. If the given integers are in descending order, return the first permutation. For example, given the integers [1, 2, 3], the next lexicographic permutation would be [1, 3, 2]. The integers are between 1 and 10^5, and there are at most 10^4 integers in the collection.

## Approach
The algorithm involves finding the first decreasing element from the right, swapping it with the smallest element greater than it to its right, and then reversing the elements to its right. This approach generates the next lexicographic permutation.

## Complexity
- Time: O(n)
- Space: O(1)

## C++ Solution
```cpp
#include <bits/stdc++.h>
using namespace std;

void nextPermutation(vector<int>& nums) {
    int i = nums.size() - 2;
    // find the first decreasing element from the right
    while (i >= 0 && nums[i] >= nums[i + 1]) {
        i--;
    }
    if (i >= 0) {
        int j = nums.size() - 1;
        // find the smallest element greater than nums[i] to its right
        while (nums[j] <= nums[i]) {
            j--;
        }
        // swap nums[i] and nums[j]
        swap(nums[i], nums[j]);
    }
    // reverse the elements to the right of i
    int left = i + 1;
    int right = nums.size() - 1;
    while (left < right) {
        swap(nums[left], nums[right]);
        left++;
        right--;
    }
}

int main() {
    vector<int> nums = {1, 2, 3};
    nextPermutation(nums);
    for (int num : nums) {
        cout << num << " ";
    }
    return 0;
}
```

## Key Takeaways
- The next lexicographic permutation can be generated by finding the first decreasing element from the right and swapping it with the smallest element greater than it to its right.
- Reversing the elements to the right of the swapped element generates the next lexicographic permutation in ascending order.
- The time complexity of this approach is O(n), where n is the number of integers in the collection.
