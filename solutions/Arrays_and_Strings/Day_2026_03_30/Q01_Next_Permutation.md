# Next Permutation

## Problem Statement
Generate the next lexicographically larger permutation of a given array of integers. If no such permutation exists, rearrange the array to its lexicographically smallest permutation. For example, given the array [1, 2, 3], the next permutation would be [1, 3, 2]. The array is 1-indexed, and the input will contain distinct integers.

## Approach
The algorithm finds the first decreasing element from the right, swaps it with the smallest larger element to its right, and then reverses the elements to the right of the swapped pair. This approach ensures the next permutation is lexicographically larger.

## Complexity
- Time: O(n)
- Space: O(1)

## C++ Solution
```cpp
#include <bits/stdc++.h>
using namespace std;

void nextPermutation(vector<int>& nums) {
    // find the first decreasing element from the right
    int i = nums.size() - 2;
    while (i >= 0 && nums[i] >= nums[i + 1]) {
        i--;
    }

    // if no such permutation exists, reverse the array
    if (i < 0) {
        reverse(nums.begin(), nums.end());
    } else {
        // find the smallest larger element to the right of i
        int j = nums.size() - 1;
        while (nums[j] <= nums[i]) {
            j--;
        }

        // swap the elements at i and j
        swap(nums[i], nums[j]);

        // reverse the elements to the right of i
        reverse(nums.begin() + i + 1, nums.end());
    }
}
```

## Key Takeaways
- The next permutation of an array can be generated by swapping the first decreasing element from the right with the smallest larger element to its right and then reversing the elements to the right of the swapped pair.
- If no such permutation exists, the array is rearranged to its lexicographically smallest permutation by reversing the entire array.
- The time complexity of the algorithm is O(n), where n is the size of the input array, since each element is visited at most twice.
